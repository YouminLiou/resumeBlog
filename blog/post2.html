如何使用 Git 和 GitHub 建立網站版本控制

← 回到部落格

發布日期：2025/5/23

1. 初始化本地 Git 儲存庫


  # 在專案根目錄執行
  git init
  git add .
  git commit -m "chore: 初次提交專案"

透過 git init 建立本地儲存庫，並第一次把所有檔案加入版控。

2. 在 GitHub 建立遠端 Repository

  1. 登入 GitHub，點「New Repository」
  2. 輸入 Repo 名稱（如 resumeBlog）、選擇公開或私有
  3. 複製 HTTPS 或 SSH 連結

3. 設定遠端並推送


  # 以 SSH 為例
  git remote add origin git@github.com:YourName/resumeBlog.git
  git branch -M main
  git push -u origin main

如果遇到「Authentication failed」，請確認你的 SSH key 是否已經加到 GitHub 帳號內。

4. 利用 Vercel 一鍵部署

  • 在 Vercel 控制台按「Import Git Repository」
  • 選擇你的 resumeBlog Repo
  • Framework 選「Other」，Output Directory 設 .
  • Deploy 完成後即可透過 your-name.vercel.app/blog 訪問

5. 小結

完整流程：git init → commit → remote add → push → Vercel deploy，實現從本地到雲端的持續部署（CI）與版本管理。
